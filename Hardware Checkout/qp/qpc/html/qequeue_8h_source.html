86/*! @brief Native QF Event Queue
87* @class QEQueue
88*
89* @details
90* This class describes the native QF event queue, which can be used as
91* the event queue for active objects, or as a simple "raw" event queue for
92* thread-safe event passing among non-framework entities, such as ISRs,
93* device drivers, or other third-party components.<br>
94*
95* The native QF event queue is configured by defining the macro
96* #QF_EQUEUE_TYPE as ::QEQueue in the specific QF port header file.<br>
97* <br>
98* The ::QEQueue structure contains only data members for managing an event
99* queue, but does not contain the storage for the queue buffer, which must
100* be provided externally during the queue initialization.<br>
101* <br>
102* The event queue can store only event pointers, not the whole events. The
103* internal implementation uses the standard ring-buffer plus one external
104* location that optimizes the queue operation for the most frequent case
105* of empty queue.<br>
106* <br>
107* The ::QEQueue structure is used with two sets of functions. One set is for
108* the active object event queue, which might need to block the active object
109* task when the event queue is empty and might need to unblock it when
110* events are posted to the queue. The interface for the native active object
111* event queue consists of the following functions: QActive_post(),
112* QActive_postLIFO(), and QActive_get_(). Additionally the function
113* QEQueue_init() is used to initialize the queue.<br>
114* <br>
115* The other set of functions, uses ::QEQueue as a simple "raw" event
116* queue to pass events between entities other than active objects, such as
117* ISRs. The "raw" event queue is not capable of blocking on the get()
118* operation, but is still thread-safe because it uses QF critical section
119* to protect its integrity. The interface for the "raw" thread-safe queue
120* consists of the following functions: QEQueue_post(),
121* QEQueue_postLIFO(), and QEQueue_get(). Additionally the function
122* QEQueue_init() is used to initialize the queue.
123*
124* <br>ote Most event queue operations (both the active object queues and
125* the "raw" queues) internally use the QF critical section. You should be
126* careful not to invoke those operations from other critical sections when
127* nesting of critical sections is not supported.
128*
129* @sa ::QEQueue for the description of the data members
130*/
