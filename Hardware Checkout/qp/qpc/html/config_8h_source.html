7/*! The preprocessor switch to disable checking assertions
8*
9* @description
10* When defined, Q_NASSERT disables the following macros #Q_ASSERT,
11* #Q_REQUIRE, #Q_ENSURE, #Q_INVARIANT, #Q_ERROR as well as
12* #Q_ASSERT_ID, #Q_REQUIRE_ID, #Q_ENSURE_ID, #Q_INVARIANT_ID, and
13* #Q_ERROR_ID do _not_ evaluate the test condition passed as the
14* argument to these macros.
15*
16* @note The notable exceptions are the macros #Q_ALLEGE and
17* #Q_ALLEGE_ID, that still evaluate the test condition, but do not
18* report assertion failures when the switch #Q_NASSERT is defined.
19*/
20#define Q_NASSERT
21
22/*! Enable the QActive_stop() API in the QF port.
23*
24* @description
25* Defining this macro enables the QActive_stop() API in a given port.
26* This feature should be used with caution, as stopping and re-starting
27* active objects **cleanly** can be tricky.
28*/
