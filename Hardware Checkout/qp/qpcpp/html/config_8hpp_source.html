4//! The preprocessor switch to disable checking assertions
5//!
6//! @description
7//! When defined, Q_NASSERT disables the following macros #Q_ASSERT,
8//! #Q_REQUIRE, #Q_ENSURE, #Q_INVARIANT, #Q_ERROR as well as
9//! #Q_ASSERT_ID, #Q_REQUIRE_ID, #Q_ENSURE_ID, #Q_INVARIANT_ID, and
10//! #Q_ERROR_ID do NOT evaluate the test condition passed as the
11//! argument to these macros.
12//!
13//! @note One notable exception is the macro #Q_ALLEGE, that still
14//! evaluates the test condition, but does not report assertion
15//! failures when the switch Q_NASSERT is defined.
16#define Q_NASSERT
17
18//! Enable the QActive::stop() API in the QF port.
19//!
20//! @description
21//! Defining this macro enables the QActive::stop() API in a given port.
22//! This feature should be used with caution, as stopping and re-starting
23//! active objects **cleanly** can be tricky.
24//!
25#define QF_ACTIVE_STOP
26
27//! The preprocessor switch to activate the constructor in QP::QEvt.
28//!
29//! @description
30//! When #Q_EVT_CTOR is defined (typically in the qep_port.hpp header file),
31//! QP::QEvt becomes a class with constructor. More importantly, the
32//! subclasses of QP::QEvt (custom events) can have non-default constructors.
33//! These constructors are then called when events are created (e.g.,
34//! with Q_NEW())
35//!
36//! @sa #Q_EVT_VIRTUAL
37#define Q_EVT_CTOR
38
39//! The preprocessor switch to activate the virtual destructor in QP::QEvt.
40//!
41//! @description
42//! This macro only works when #Q_EVT_CTOR is also defined. When also
43//! #Q_EVT_VIRTUAL is defined (typically in the qep_port.hpp header
44//! file), QP::QEvt becomes a class with a constructor and a virtual
45//! destructor. More importantly, the subclasses of QP::QEvt (custom events)
46//! can have (virtual) destructors. These destructors are then invoked
47//! before recycling the event with QP::QF::gc().
48#define Q_EVT_VIRTUAL
49
50//! The preprocessor switch to activate the QS software tracing
51//! instrumentation in the code
52//!
53//! @description
54//! When defined, Q_SPY activates the QS software tracing instrumentation.
55//! When Q_SPY is not defined, the QS instrumentation in the code does
56//! not generate any code.
